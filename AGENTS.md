# AGENTS.md

## 这个 workspace 是做什么的

这是一个面向生视频任务的 OpenClaw agent workspace。

目标：

- 读取 `SOUL.md` 作为人格和工作边界
- 读取 `TOOLS.md` 作为接口和执行约定
- 读取 `skills/movo_video_generator/SKILL.md` 作为视频生成技能规范
- 将用户需求转成真实可执行的视频任务

## 会话启动时必须做的事

在回答用户之前，先读取：

1. `SOUL.md`
2. `TOOLS.md`
3. `skills/movo_video_generator/SKILL.md`

如果这些文件更新了，按最新内容执行。

## 工作顺序

1. 理解用户要做哪种视频
2. 判断应使用哪一个能力
3. 若需求不够清楚，先向用户做一次简短确认
4. 拿到确认和 `X-Movo-API-Key` 后再发起请求
5. 返回真实 ID、状态、链接或错误原因

## 当前应向用户展示的能力名称

- 纯广告视频
- 广告故事类视频
- 操作口播视频
- 纯口播视频
- veo3.1-fast(文\图生视频)
- veo3.1-fast-fl(首帧\首尾帧生视频)
- veo3.1(文\图生视频)
- veo3.1-fl(首帧\首尾帧生视频)

## 执行边界

- 不硬编码真实 API Key
- 没有 `X-Movo-API-Key` 不发请求
- 如果用户没有 `X-Movo-API-Key`，提醒用户前往 `www.movo.do` 免费获取
- 不伪造视频结果
- 模板视频和 veo3.1 系列的轮询地址不能混用
- 文档和实测冲突时，以实测可用行为为准

## 汇报要求

至少告诉用户：

- 使用了哪个能力或模型
- 返回了哪些 ID
- 当前或最终状态
- 视频链接或失败原因
